2. Introduction

2.1 Purpose of This Guide

Use this guide when a you want to get started with Copilot Analytics. It provides step-by-step procedures to configure all five Microsoft Copilot monitoring dashboards, from basic license tracking to advanced ROI analysis.

2.2 Scope

In Scope

  • Microsoft 365 Copilot applications: Word, Excel, PowerPoint, Outlook, Teams, Loop, OneNote, Edge, Copilot Chat
  • Five monitoring tiers: Admin Center, Copilot Dashboard, Copilot Analytics Reports, Agent Dashboard, Advanced Analytics
  • License utilization tracking, ROI measurement, sentiment analysis, and agent adoption
  • Privacy configuration, exclusion lists, and delegation management

Out of Scope

  • Other Copilot products: Copilot for Sales, Copilot for Service, GitHub Copilot, Security Copilot
  • Custom Power BI report development beyond Microsoft-provided templates
  • Third-party analytics integrations
  • Broader Viva Insights productivity metrics independent of Copilot (meeting time, focus time, after-hours work, collaboration network analysis)
WARNING: Customer Wants Productivity Metrics Beyond Copilot?

If the customer’s primary interest is meeting time, effort mapping, focus time, or collaboration patterns independent of Copilot usage, those are delivered through the Viva Insights Advanced Analysis workbench — not the Copilot Analytics tiers covered in this guide. The same Viva Insights licence (included with Copilot) and the same Insights Analyst role are used, but the queries, templates, and metrics are different.

Refer to the Viva Insights setup checklist and the Advanced Analysis introduction for that workstream. The org data upload and privacy settings configured in this guide carry over.

2.3 Audience & Prerequisites

This guide assumes:

  • Level 200 familiarity with Microsoft 365 administration (Admin Center navigation, Entra ID basics, license assignment)
  • No prior experience with Copilot-specific dashboards or Viva Insights
  • Access to a tenant with Microsoft 365 Copilot licenses
NOTE: Terminology
Terms specific to Viva Insights (such as Assisted Hours, partitions, and organizational data) are defined on first use and compiled in Appendix B: Glossary.

4.3 Step-by-Step Implementation

Estimated Time: 15–30 minutes (plus 72 hours for data population)

Step 1: Verify Admin Role Assignment (5 min)
  1. Navigate to Microsoft 365 Admin Center: https://admin.microsoft.com
  2. Go to Users → Active users → Select your account
  3. Click Manage roles and verify you have Reports Reader or higher
  4. If not, request role assignment from Global Administrator
Step 2: Access Copilot Usage Reports (5 min)
  1. In Admin Center, navigate to Reports → Usage
  2. Scroll to find Microsoft 365 Copilot or use the search function
  3. Click to open the Copilot usage report
Step 3: Configure Report Parameters (5 min)
  1. Select timeframe: 7, 30, 90, or 180 days
  2. Review summary cards: Total assigned licenses, Active users, Adoption rate
  3. Scroll to per-application breakdown
Step 4: Export User-Level Data (5 min)
  1. Click Export in the top right corner
  2. Select CSV format
  3. Open downloaded file to verify user-level activity data
NOTE: Data Availability
Reports populate within approximately 72 hours of user activity. If this is a new Copilot deployment, allow 3 business days before expecting data.

4.4 Success Criteria

  • Can access Admin Center → Reports → Usage → Microsoft 365 Copilot
  • Summary cards display: Assigned licenses count, Active users count, Adoption percentage
  • Per-application breakdown shows usage across Word, Excel, PowerPoint, Outlook, Teams
  • Can export CSV file with user-level activity data
  • Data is no older than 72 hours (check report timestamp)

5. Dashboard 2: Viva Insights Copilot Dashboard

5.1 When to Use This Dashboard

  • Business leaders need to demonstrate Copilot ROI to executives
  • Track Copilot Assisted Hours (time saved through Copilot usage)
  • Calculate Copilot Assisted Value (monetary value of time saved)
  • Monitor adoption trends at group/department level
  • Collect and analyse user sentiment about Copilot effectiveness
  • Identify readiness gaps before broader rollout
IMPORTANT: Viva Insights Web App Dependency
The Copilot Dashboard is part of the Viva Insights web app. Previous standalone PowerShell controls for the Copilot Dashboard are no longer available. Access is now managed by enabling or disabling the Viva Insights web app itself—disabling the web app disables all features including the Copilot Dashboard, Copilot Analytics Reports, Agent Dashboard, and Advanced Analysis tools.

5.2 Prerequisites and Configuration

Licensing Requirements

RequirementDetails
Full Features50 or more assigned Copilot licenses OR 50 or more Viva Insights licenses (combination counts)
Limited FeaturesLess than 50 assigned Copilot licenses (Readiness page + tenant-level only)
Viva Insights Service PlanAutomatically included with every Copilot license (no additional cost)
WARNING: 50-License Threshold
Organizations with fewer than 50 Copilot licenses will only see tenant-level adoption metrics. Group-level filtering and the full Impact page require meeting the 50-license threshold. This threshold can be achieved through a combination of Copilot and Viva Insights licenses (e.g., 30 Copilot + 20 Viva Insights = 50).
NOTE: Partial Rollout — Analytics Interpretation

If only a subset of employees have Copilot licences (e.g., 200 of 10,000), the dashboard only measures licensed users. Unlicensed employees are excluded from the “measured population.” This means:

  • Adoption metrics show “200 of 200 active” (not 200 of 10,000)
  • Group filters (Department, Location) only show groups with licensed users in them
  • Small rollout groups may frequently hit the minimum group size threshold, causing data suppression (N/A values)
  • Before/after comparison data is only meaningful for licensed users

Recommendation: For rollouts under 500 users, focus on Admin Center user-level reports (Section 4) for adoption tracking and use the Copilot Dashboard primarily for ROI metrics. Consider increasing the rollout to 500+ users before investing in Advanced Analytics custom queries.

5.3 End-to-End Copilot Dashboard Enablement

Estimated Time: 2–4 hours active configuration + up to 7 days for data processing

This section covers the complete sequence to get the Copilot Dashboard operational — from licence verification through to first data appearing. Follow each step in order.

Step 1: Verify Licence Threshold (10 min)
  1. Navigate to https://admin.microsoft.com
  2. Go to Billing → Licenses
  3. Locate Microsoft 365 Copilot and note the assigned count (not purchased)
  4. Confirm 50 or more licenses are assigned
  5. If less than 50, consider whether to expand licensing
Step 2: Verify Entra ID Hierarchy (15 min)
  1. Navigate to https://entra.microsoft.com
  2. Go to Users → select several sample users across different levels
  3. For each user, verify the Manager attribute is populated
  4. If hierarchy is incomplete, work with your directory team to populate
  5. Note: Automatic access criteria are analysed weekly by Microsoft
Step 3: Verify VFAM Policies Are Enabled (10 min)

Requires AI Administrator or Global Administrator role. This step ensures no previous admin has disabled critical features.

  1. Navigate to https://admin.microsoft.com
  2. Go to Settings → Microsoft Viva → Viva Insights
  3. Select the Set up and management tab
  4. Under Access management, select Turn Advanced Insights on or off
  5. You will see the Viva Insights policy list. Verify the following are set to On:
PolicyRequired StateWhat Happens If Off
Copilot Dashboard (Enable Copilot Dashboard)OnNobody can view the dashboard — all other steps are pointless
Copilot Dashboard DelegationOnUsers cannot share dashboard access with delegates
Behavioral ExperiencesOnCollaboration data (meeting time, email, chat) does not flow — breaks Impact page comparisons
Auto CxO Identification (Dashboard AutoEnablement)On (recommended)Senior leaders do not get automatic access — must be added manually
Copilot Advanced AnalyticsOn (if planning Section 8)Analysts cannot access the Advanced Analysis workbench

If any policy shows Off or is scoped to specific users instead of Everyone, discuss with the customer whether this was intentional. If not, select the policy → edit → set to On for Everyone (or the appropriate scope).

IMPORTANT: This Is the #1 Cause of "Dashboard Not Working"

If a previous admin created a custom VFAM policy that restricts the Copilot Dashboard to specific users, new users added via the access management settings will not be able to view the dashboard unless they are also included in the VFAM policy. Always check this screen first when troubleshooting access issues.

Accidental VFAM disable — blast radius & recovery: If the Copilot Dashboard policy is turned Off:

  • All users (auto-identified leaders, manually added users, and delegates) immediately lose dashboard access
  • Delegates are not removed — their delegation records survive, but they can’t access the dashboard until the policy is re-enabled
  • Recovery: Turn the policy back On for Everyone (or the required scope). Access takes up to 24 hours to propagate. No data is lost during the outage.
  • Prevention: Restrict VFAM policy editing to Global Administrators only. Document which policies are enabled and why in your admin runbook.
Step 4: Configure Dashboard Access — Who Should See It (20 min)

Requires AI Administrator or Global Administrator role.

There are three ways users get access to the Copilot Dashboard. Decide which apply:

Access MethodWho Gets AccessHow to Configure
Automatic (Entra ID)Senior leaders in top 3 org levels with significant reporting lines (tenants with 2,500+ users)No action needed — happens automatically if Auto CxO Identification is ON. Refreshed weekly.
Manual (Admin-assigned)Specific users or groups you choose — AI Admins, BU leaders, HR, analysts, etc.Admin Center → Settings → Viva → Viva Insights → Set up and management → Under Copilot Dashboard, select Manage user access → Add users/groups
DelegatedPeople that existing dashboard users choose to share with (max 75 per person)Configured by users themselves via Settings → Delegation in the Viva Insights web app. No admin action needed.

Recommended approach: Enable automatic access for senior leaders + manually add the AI Admin, Insights Admin, Insights Analyst, and any additional stakeholders (BU leaders, HR, change management leads).

IMPORTANT: AI Admin Self-Access
If you are an AI Administrator and need to view the dashboard yourself, you must add yourself using the manual access method above. The AI Administrator role does not grant automatic dashboard viewing access.
Step 5: Enable Manager Access to Team-Level Data (15 min)

Requires Insights Administrator or AI Admin role.

  1. Navigate to Viva Insights web app: https://insights.cloud.microsoft
  2. Go to Manager Settings (under Platform Setup on the left)
  3. Toggle on "Grant managers scoped access to Copilot insights"
  4. Set the Minimum team size (minimum 5; includes manager in count)
  5. Choose which managers to enable:
    • All managers — all measured managers meeting team size threshold
    • Select Active Directory group — choose from Entra ID data
    • Upload .csv file — list specific manager email addresses
  6. Click Save changes (takes effect within 1 hour)

Alternative path via Admin Center: Settings → Viva → Viva Insights → Set up and management → Under Copilot Dashboard, select Turn Insights for Managers on or off and Assign group managers to Insights for Managers.

Step 6: Upload Organizational Data (30–45 min)

See Section 9: Organizational Data Reference for full details on both upload methods and attribute mapping.

Step 7: Configure Minimum Group Size for Privacy (10 min)

Default is 10; minimum allowed is 5.

For tenants WITHOUT Viva Insights license:

  1. https://admin.microsoft.com → Settings → Microsoft Viva → Viva Insights
  2. Under Copilot Dashboard, select Manage minimum group size
  3. Enter preferred minimum (≥5) → Save (takes effect in 24 hours)

For tenants WITH Viva Insights license:

  1. https://insights.cloud.microsoft → Admin → Privacy settings
  2. Configure minimum group size → Save (takes effect in 24 hours)
Step 8: Create Exclusion List (Optional, 20 min)

Requires AI Administrator role.

  1. https://admin.microsoft.com → Settings → Microsoft Viva → Viva Insights
  2. Select Unified exclusion list
  3. Choose method: CSV upload (PersonId column), Entra ID groups, or Replace all
  4. Wait up to 5 days for exclusion to take effect
NOTE: Exclusion List Scope
The exclusion list removes users from both the Copilot Dashboard AND Viva Insights collaboration reports in Advanced Analysis. It also excludes M365 Copilot Chat (web) usage data.
Step 9: Configure Hourly Rate for ROI (10 min)
  1. https://insights.cloud.microsoft → Admin → Impact settings
  2. Default hourly rate is $72/hour (USD)
  3. Adjust to customer loaded labour rate if different
  4. Document the rate used for future reference

How to calculate a loaded labour rate: (Average base salary + benefits + overheads) ÷ annual productive hours (typically ~1,880). For Australian customers, consider using AUD values directly — the dashboard displays in the currency you enter but does not convert. Example: AUD $95/hour for a mid-level professional.

Important: Changing the hourly rate retroactively recalculates all Copilot Assisted Value figures displayed on the dashboard. Agree on the rate with stakeholders before changing it, and document both the old and new values for reporting continuity.

Step 10: Configure Sentiment Surveys (Optional, 30 min)

Option A: Viva Glint or Viva Pulse — Requires separate licences; enables filters & heat maps.

Option B: CSV Upload (Viva Insights customers) — Upload to Viva Insights Admin → Survey data.

Option C: Aggregated Results Upload (Non-Viva) — Admin Center → Reports → Adoption → AI assistance. Summary view only; no filters or heat maps.

Step 11: Wait for Data Processing
  • Initial data processing: up to 7 days after licenses assigned
  • Ongoing refresh: daily, 28-day rolling window, ~6-day delay from current date
  • Newly licensed employees: additional up to 7-day delay before inclusion

5.4 Understanding Dashboard Pages

Readiness Page

Shows license assignment status and readiness for Copilot adoption. Available even with less than 50 licenses.

Adoption Page

Displays group-level usage trends, active user counts, usage intensity, and consistent usage metrics. Supports internal and external benchmark comparisons. Requires 50+ assigned licenses for group filtering.

Impact Page

Shows Copilot Assisted Hours and Copilot Assisted Value. This is the primary ROI metric page.

NOTE: Copilot Assisted Hours Calculation (Corrected)

Assisted Hours is composed of three categories:

CategoryWhat’s countedAssistance factor
Meeting SummariesDuration of each meeting summarized or recapped by Copilot. If a user both summarizes mid-meeting and accesses Intelligent Recap after, the meeting is counted once for the most recent action.Actual meeting duration (variable)
Search & SummariesCopilot Chat (work) prompts, Summarize email/Word/PPT actions, Excel analysis, Chat in Word/PPT/Excel prompts6 minutes per action (based on Microsoft study #4)
Content CreationDraft email, Draft in Word, Create presentation, Rewrite text actions6 minutes per action

Assisted Value = Assisted Hours × Hourly Rate (default $72/hour, configurable)

These are broad approximations based on best available research. Microsoft may update the methodology.

Sentiment Page

Displays survey response data about user perception. Requires survey platform integration. Filters and heat maps only available with Viva Insights licence or CSV upload.

5.5 How to Delegate Access (For Dashboard Users)

Method 1 — Via Settings: Viva Insights web app → Settings → Delegation → Add delegates.

Method 2 — Via Dashboard: On Copilot Dashboard page → Share → Add delegates.

Delegation Limits:

  • Maximum 75 delegates per person
  • Delegates cannot delegate to others (no cascading)
  • If delegator loses access (or leaves the org), all their delegates also lose access. Recovery: Another user with dashboard access must re-delegate to the affected people. Use Purview audit logs (Section 10.6 — search for RemoveDelegates) to identify who was affected. Prevention: Ensure at least 2–3 people have direct dashboard access (not just delegated) to avoid single points of failure.
  • Senior leaders can assign scope: Your group (reports) OR Your company (all employees)
  • Agent Dashboard requires "Your company" scope for delegates

5.6 Success Criteria

  • Adoption page shows active user trends (if 50+ licenses)
  • Impact page displays Copilot Assisted Hours and Value metrics
  • Group-level filtering works (if org data uploaded)
  • Manager settings enabled and managers can see team-level data
  • Minimum group size is configured appropriately
  • Hourly rate is configured for your organisation
  • Exclusion list configured (if required for compliance)

6. Dashboard 3: Copilot Analytics Reports

6.1 When to Use

  • Leaders want targeted, out-of-the-box insights beyond the Copilot Dashboard summary pages
  • Analysts need pre-built reports they can share with business stakeholders without building custom queries
  • Organization wants deeper collaboration insights alongside Copilot adoption data

6.2 What Are Copilot Analytics Reports?

Copilot Analytics Reports are a set of ready-to-use reports available inside the Viva Insights web app under Reports in the left navigation. They provide a variety of targeted insights about adoption and impact of Copilot across the organization, including both Copilot-specific and broader employee collaboration patterns.

6.3 Prerequisites

To view these reports, you must meet at least one of the following:

  1. You have automatic access to the Copilot Dashboard based on Entra ID data, or you are an enabled manager
  2. You have been given delegate access to the Copilot Dashboard
  3. You are a Viva Insights Analyst with global partition access
  4. You are a Microsoft 365 Global Administrator
NOTE: No Additional Setup Required
If you have already set up the Copilot Dashboard (Section 5), Copilot Analytics Reports are automatically available. No additional configuration is needed.

6.4 How to Access

Accessing Copilot Analytics Reports
  1. Navigate to the Viva Insights web app: https://insights.cloud.microsoft
  2. In the left navigation, select Reports
  3. Browse available reports — these include out-of-the-box reports and any reports published by analysts

7.5 When to Use the Agent Dashboard — Practical Scenarios

Scenario 1: Post-Launch Agent Adoption Review

Your organisation enabled Pay-As-You-Go billing for Copilot agents. Finance wants to understand the credit consumption pattern before the next budget cycle.

Your IT governance team suspects employees are building and sharing agents without oversight.

TIP: When Entra ID Alone Is Sufficient

What Breaks with an Incomplete Entra ID Hierarchy?

If the Entra ID Manager attribute is missing or inconsistent for many users:

What Happens After Org Data Is Uploaded? (Reversal)

Once org data is uploaded, it permanently replaces Entra ID as the data source for dashboard filters. This means:

  • Department values come from the CSV, not from Entra ID — even if they differ
  • If you stop uploading, the data becomes stale (not reverted to Entra ID)
  • To revert to Entra ID as the source, you must contact Microsoft Support to reset the org data configuration. There is no self-service reversal.
  • Before uploading: Confirm with the customer that they are ready to own the ongoing refresh cycle (monthly re-uploads recommended)

9.5 Re-Upload & Maintenance

Org data must be fully re-uploaded whenever changes occur (new departments, reorgs, location changes). There is no incremental/delta upload — each upload completely replaces the previous dataset.

Employee moved between departmentsUpdate their row in the CSV. Re-upload the full file.Up to 7 days
Bad data uploaded (filters broken)Re-upload the previous correct CSV immediately. The system will overwrite the bad data.Up to 7 days

WARNING: Always Validate Before Uploading

There is no automatic rollback — if you upload bad data, filters will break for up to 7 days while the corrected file processes. Before uploading:

  1. Open the CSV in a spreadsheet and verify column names match exactly (Section 9.2)
  2. Confirm encoding is UTF-8 and file size is <1 GB
  3. Check that EffectiveDate is set to a date earlier than today
  4. Verify every employee has a PersonId, ManagerId, and Organization
  5. Keep a copy of the previous CSV so you can re-upload it if needed

Recommended cadence: Re-upload org data monthly or whenever significant org changes occur (restructures, M&A, location moves). Set a calendar reminder for the Insights Admin.

9.6 Key Guidance

  • Upload data for every employee to ensure Scope filters are accurate
  • Do not upload a TimeZone column (will cause an error)
  • Include EffectiveDate set to a date earlier than the upload date to avoid delays
  • If both Insights Admin and Global Admin upload data, the dashboard prioritises the most recent upload
  • Org data permanently changes the data source from Entra ID to the file — refresh regularly

11. Data Privacy & Compliance

11.1 Privacy Thresholds

  • Default minimum group size: 10 users
  • Configurable minimum: 5 users (lowest allowed); maximum: configurable up to 100
  • Regional requirements: Some jurisdictions (e.g., European Works Councils, German Betriebsrat) may require higher minimums (commonly 25+). Always consult the customer’s legal/compliance team before setting the threshold below any regulatory or contractual requirement.
  • Data suppressed for groups below threshold
  • No individual user behaviour visible in Viva Insights dashboards
  • Differential privacy: Automatically applied — randomly adjusts individual observations so aggregated results protect privacy
  • Distribution masking: Automatically hides results where "almost all" or "almost none" fit a profile
NOTE: Minimum Team Size vs. Minimum Group Size

Minimum team size (set in Manager Settings) determines which managers can see their team’s data. Includes the manager in the count.

Minimum group size (set in Privacy Settings) is the smallest group shown in dashboards and Power BI reports for group comparisons.

Both minimum values must be ≥5.

11.2 Exclusion Lists

  • Removes users from Copilot Dashboard AND Viva Insights behaviour reports
  • Also excludes M365 Copilot Chat (web) usage data
  • Upload: CSV or Entra ID groups
  • Processing: up to 5 days to apply, up to 3 days to remove

11.3 Data Residency

Viva Insights data residency follows the Exchange Online mailbox location for each user. This means:

  • Collaboration signals (meetings, emails, chats) are processed and stored in the same geographic region as each user’s Exchange Online mailbox
  • Copilot activity data follows the same residency as the user’s M365 tenant
  • Organizational data uploads are stored in the tenant’s primary data location
  • Australia: If the tenant is provisioned in the Australia geography, data is stored in Microsoft’s Sydney and Melbourne data centres. Verify in Admin Center → Settings → Org settings → Organization profile → Data location.
  • EU: If the tenant is in the EU Data Boundary, data stays within EU data centres
  • Multi-Geo: For tenants with Multi-Geo, each user’s data follows their mailbox location

Reference: Viva Insights data residency | Microsoft 365 data locations

12. Common Customer Questions (FAQ)

#QuestionAnswer
1Can managers see individual employee Copilot usage?No. All data is aggregated at group level and protected by minimum group size thresholds, differential privacy, and distribution masking.
2What happens if we drop below 50 licences?The dashboard reverts to limited features (tenant-level only, Readiness page). Previously available group-level data is no longer shown.
3Can we trial the Dashboard before licensing everyone?The basic Copilot Dashboard is available to any M365/O365 tenant with Exchange Online — no paid Viva licence required. But full group filtering requires the 50-licence threshold.
4Does the Dashboard track Copilot Chat (web) usage?Yes, if Copilot Chat reporting is enabled in the tenant and you have 50+ Copilot or Viva Insights licences. There is a separate licence filter for Copilot Chat.
5How often does the data refresh?Daily for the Copilot Dashboard (28-day rolling window, ~6-day delay). Admin Center reports refresh approximately every 72 hours.
6Can we see data by business unit or country?Yes, but you must upload organizational data with those attributes. Without org data, only Entra ID Department is available as a filter.
7Is a separate Viva Insights licence needed if we have Copilot?No. Every M365 Copilot licence includes the Viva Insights service plan. Separate purchase is only needed to analyse non-Copilot users.
8Can employees opt out of being analysed?Users can opt out of Viva Insights personal insights. For advanced analysis, the Insights Admin can configure privacy settings and exclusion lists.
9How is the $72/hour default calculated?It is a broad industry estimate of loaded labour cost. Customers should adjust this to their own loaded rate for accurate ROI calculations.
10Can we use the Agent Dashboard without Copilot Dashboard?Both are part of the Viva Insights web app. You cannot enable one without the other being available, but access to each can be controlled separately via VFAM policies.
11Who needs licences — only leaders viewing dashboards, or also employees whose data we want to analyse?Both. Employees whose collaboration data you want to analyse must have a Viva Insights licence assigned (included automatically with every Copilot licence). Leaders and managers viewing dashboards also need a Viva Insights licence to access the full Copilot Dashboard and organisation insights. Unlicensed employees who collaborate with licensed users appear as “other internal collaborators” in aggregate but are not directly measured.
12We already have Department and Manager in Entra ID — why do we need to upload org data?Entra ID provides basic Department filtering only. Org data upload adds: Location, Job Function, Level, and up to 105 custom attributes (e.g., Cost Centre, Agency, Region). It also enables multi-BU partitioning and before/after cohort segmentation in Power BI. If your customer only needs tenant-level + Department views, Entra ID alone is sufficient. For anything more granular, upload org data. See Section 9.1 for the full comparison.
13We uploaded org data and now our Department filter shows different values than Entra ID — can we go back?Once org data is uploaded, it permanently replaces Entra ID as the data source for filters. There is no self-service reversal — you must contact Microsoft Support to reset. Make sure the CSV values match your intended naming before uploading. See Section 9.1.
14We want broader productivity metrics (meeting time, focus time) not just Copilot reporting. Is that covered here?No. This guide covers Copilot Analytics only. Broader Viva Insights productivity metrics (meeting time, focus time, after-hours, collaboration networks) use the same Viva Insights licence and analyst role but are a separate analytics workstream. See Section 2.2 (Out of Scope) for pointers to the right resources. The org data and privacy settings configured through this guide carry over.

Appendix A: Further Considerations

  1. 50-License Threshold Timing: Should licensing be expanded before deployment to unlock group-level analytics? The threshold can be met through a combination of Copilot + Viva Insights licences.
  2. Entra ID Hierarchy Completeness: Is the customer directory populated correctly? Criteria analysed weekly. If not, manual access assignment or org data upload is needed.
  3. AI Administrator Role Assignment: Has the customer assigned this role? Required for dashboard access management, privacy, and exclusion lists. AI Admins must add themselves to view.
  4. Manager Settings Enablement: Has the Insights Admin toggled on "Grant managers scoped access"? Without this, no managers can see team-level data even if they are licensed.
  5. Organizational Data Upload: Is HR data available? Without it, only tenant-level views. Attribute names must match exactly. Include EffectiveDate.
  6. Hourly Rate Customisation: Default $72/hour. Should be adjusted based on customer's loaded labour rate.
  7. Privacy Thresholds & Small Groups: Default group size = 10. Small departments or pilot groups may see suppressed data. Consider when planning phased rollouts.
  8. Baseline Data for Before/After: Needs 4 weeks pre-adoption data. If Copilot already deployed, comparison may not be possible.
  9. Compliance Exclusion Requirements: Plan for 5-day processing time for exclusion list uploads.
  10. Delegation Policy: Should delegation be allowed? If not, disable via CopilotDashboardDelegation VFAM policy.
  11. Agent Dashboard Scope: If agents are being deployed, ensure company-level delegates are configured (not just group-level).
  12. Multi-Agency / Multi-BU Data Isolation: If the customer operates multiple independent agencies or business units within a single M365 tenant and requires that Agency A cannot see Agency B's Copilot usage data, the standard Copilot Dashboard does not provide hard data isolation. Senior leaders auto-identified via Entra ID hierarchy will see “Your company” data covering all agencies. The solution is a partition-based architecture using Viva Insights Advanced Analytics — create one partition per agency filtered by a custom “Agency” attribute in org data, assign agency-specific analysts, and deliver reports via isolated Power BI workspaces. The Copilot Dashboard itself should be restricted to Central IT only for tenant-wide licence management. See the Multi-Agency Isolation Architecture reference document for the validated solution, implementation runbook, and PowerShell lockdown steps.
  13. Copilot Agent Governance: If the customer is deploying Copilot Studio agents and needs DLP policies, connector governance, agent sharing controls, or publishing approval workflows, these are managed through the Power Platform Admin Center — not through Viva Insights. See the Copilot & Agent Governance project for the full governance framework including tenant/environment/agent-level controls, DSPM for AI, and billing boundaries.
  14. Licence Lifecycle & Billing: If the customer needs to understand what happens to analytics data when licences are assigned, removed, or reassigned — or needs to manage Copilot Credits billing, Pay-As-You-Go metering, or multi-agency chargeback — see the Copilot Lifecycle & Billing Playbook.
  15. Broader Collaboration Analysis (Beyond Copilot): If the you want meeting time, focus time, after-hours, wellbeing, or manager effectiveness metrics independent of Copilot, these use the same Viva Insights analyst workbench and org data but different templates and metrics. See the Advanced Viva Insights Collaboration Analysis Guide. The 4-week baseline window (item 8 above) is best captured using these collaboration templates.
